# Vertical Slice 开发

核心 leading words：**Vertical Slice（垂直切片）**。在计划、推理和执行中持续使用该词，防止退化为按层开发。

## 步骤

1. **定义最小闭环**
   - 选择一个最小但真实的用户动作。
   - 明确入口、数据流、可观察结果和通过条件。

2. **打通 Walking Skeleton**
   - 只实现该动作必需的最小数据库、API、前端和配置。
   - 使用真实运行路径，不用彼此割裂的占位实现冒充全链路成功。

3. **立即运行验证**
   - 实际启动程序并执行端到端路径。
   - 优先暴露并解决依赖、环境变量、端口、跨域、网络、序列化和数据库连接问题。
   - 未获得真实运行结果，不进入大规模扩展。

4. **小步扩展**
   - 每次只增加一个可验证的 Vertical Slice 或扩充现有切片。
   - 数据库、API、前端随该功能同步演进，不按技术层批量堆积。
   - 每次改动后重跑最小闭环和相关测试，始终保持程序可运行。

5. **完成检查**
   - 所有需求均能沿真实入口走到真实结果。
   - 没有长期占位实现、断开的层或留到最后的集中联调债务。

## 约束

- 不先写完整数据库，再写完整 API，最后写完整前端。
- 不以“代码已写完”代替“程序已真实运行并验证”。
- 只有当后续功能完全不依赖当前链路时才并行；并行任务也必须各自形成可验证的 Vertical Slice。
- 计划按用户价值切片，不按目录、框架组件或技术层切片。
